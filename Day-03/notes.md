# Day 03 - n8n Nodes

## Notes

- Nodes are the fundamental building blocks of n8n workflows.
- Every workflow consists of multiple connected nodes.
- Each node performs a specific task within a workflow.
- Nodes receive input data, process it, and produce output data.
- Workflows are created by connecting nodes together.

### Types of Nodes

- Trigger Nodes
  - Start the workflow.
  - Define when the workflow should execute.

- Regular Nodes
  - Process, transform, or manipulate data.
  - Perform actions within the workflow.

### Node Configuration

- Each node has configurable parameters.
- Nodes can be customized based on the required task.
- Input and output data can be viewed during execution.
- Node settings help control behavior and execution.

### Examples of Nodes

- Manual Trigger
- Schedule Trigger
- HTTP Request
- Google Sheets
- Set Node
- Function Node

### Key Learnings

- Nodes are the core components of n8n workflows.
- Different nodes serve different purposes.
- Data flows from one node to another.
- Understanding nodes is essential before building workflows.
