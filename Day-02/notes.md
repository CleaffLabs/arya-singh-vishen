# Day 2 - APIs and Webhooks

## Overview

The session focused on understanding how applications communicate with each other using APIs and webhooks. These concepts form the foundation of workflow automation and integrations.

## Topics Covered

* What is an API?
* API Requests and Responses
* API Endpoints
* HTTP Methods
* What is a Webhook?
* API vs Webhook

## Key Concepts

### API (Application Programming Interface)

An API allows different software applications to communicate and exchange data with each other.

Example:

* A weather application requests data from a weather service API.
* The API returns the requested weather information.

## API Communication Flow

Application
→ Request
→ API Server
→ Response
→ Application

## Common HTTP Methods

### GET

Retrieve information from a server.

### POST

Create new data on a server.

### PUT

Update existing data.

### DELETE

Remove data.

## API Response

An API response usually contains:

* Status Code
* Requested Data
* Error Information (if any)

### Common Status Codes

* 200: Success
* 201: Created
* 400: Bad Request
* 404: Not Found
* 500: Internal Server Error

## What is a Webhook?

A webhook is an event-driven mechanism that automatically sends data from one application to another when a specific event occurs.

Example:

* A user submits a form.
* A webhook is triggered.
* Data is automatically sent to another system.

## API vs Webhook

### API

* Request-based communication
* Client initiates the request
* Used when data is needed

### Webhook

* Event-based communication
* Server initiates the notification
* Used for real-time updates

## Importance in Automation

APIs and webhooks enable:

* Application integration
* Data transfer between systems
* Automated workflow triggers
* Real-time notifications

## Key Learnings

* APIs allow software systems to communicate.
* Requests and responses are the core components of API interactions.
* Webhooks provide real-time event-driven communication.
* APIs and webhooks are fundamental building blocks of automation workflows.
