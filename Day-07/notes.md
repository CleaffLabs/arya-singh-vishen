# Day 7 - Collaboration

## Overview

Learned how collaboration and access management work in n8n, including workflow sharing, credential sharing, and user roles.

## Topics Covered

* User Management
* Workflow Sharing
* Credential Sharing
* n8n API
* Security Audits
* Community & Templates

## Key Concepts

### User Roles

* **Owner** – Full access to the instance.
* **Admin** – Manages workflows and users.
* **Member** – Access only to assigned workflows.

### Workflow Sharing

Workflows can be shared with team members while controlling editing permissions.

### Credential Sharing

Credentials can be shared securely without exposing API keys or sensitive information.

### Community & Templates

* Community forum for support and discussions.
* Template library for pre-built workflows.

## Key Learnings

* Workflows and credentials can be shared securely.
* User permissions control access levels.
* Sensitive credential data remains hidden.
* Templates help speed up workflow development.
